# Simple replay attack on Scalable SGX

This is a PoC for a replay attack inside the EPC. It assumes a cooperative
debug scenario where a victim and attack enclave are manually synchronized
through a CLI interface.

This PoC consists of two enclaves, the victim and the attacker:

The victim enclave will:
* Initialize its buffer with all zeros.
* Overwrite the buffer with all ones.
* Print the final buffer.

The attacker enclave aims to:
* Capture the victim buffer while it contains all zeros.
* Replay the captured zeroed buffer after the victim has overwritten its buffer.

## Build

### Linux SGX driver
First, build and install the modified out-of-tree SGX driver by applying the
patches in `../patches/linux-sgx-driver.patch`. These patches ensure the EPC
pages are allocated at physical addresses that are not affected by the
interposer, thus preventing crashes when enabling the interposer. Additionally,
they provide an interface through which EPC pages can be allocated at
chosen physical addresses.
Make sure the in-tree SGX driver is disabled by adding `nosgx` to the kernel
command line arguments. (While similar modifications can also be made to the
in-tree driver, we use the legacy SGX driver).

### SGX-Step

Build and install the modified SGX-Step library. We currently only use this
library to translate virtual to physical addresses.

### Victim and attacker enclaves

Both victim and attacker enclaves can be build using `make all`. This will also
build the BadRAM kernel module and helper library.

```
make clean
make all
make insmod
```

## Use
  
**Step 1: Define the Alias Function**

Start by defining the alias function in the aliases.csv file. This file will be
used to calculate the alias pa to the victim buffer.

**Step 2: Start the Victim Enclave**

Launch the victim enclave, which will allocate a buffer that the attacker enclave will later capture and replay. During this step, the physical address  of the victim's buffer and its alias pa will be printed.

```
sudo ./victim-enclave/build/binaries/app <path to aliases.csv>
```

**Step 3: Start the Attacker Enclave**

Start the attacker enclave with the alias pa as an argument. The modified SGX driver will allocate the attacker buffer at the specified pa, which is an alias of the victim buffer.

Note: To ensure the attacker buffer is allocated correctly, you must know the EPC allocation order for the attacker enclave. If the buffer is the `n`th allocation, set `alloc_nb` on line 83 of the code to `n-1`. Determining the correct value for `alloc_nb` may require some trial and error.

```
sudo ./attacker-enclave/build/binaries/app <alias pa to victim buffer>
```


## Example output

If the replay attack was successful, certain cache lines within the victim buffer should be zero. Note that not all cache lines within the EPC page are affected due to the interleaving between different memory channels.
